# Changes - April 21, 2026

## 1. Fixed SL Verification Endpoint (CRITICAL)

**Problem:**
- Bot was checking `/fapi/v1/openOrders` for stop loss orders
- SL orders are placed via `/fapi/v1/algoOrder` and only appear in `/fapi/v1/openAlgoOrders`
- This caused false "MISSING SL DETECTED" warnings on every trade
- Bot attempted to place duplicate SL orders

**Solution:**
- Added `get_algo_open_orders()` method in `src/order_executor.py`
- Updated `verify_and_place_missing_sl()` to check algo orders endpoint: `/fapi/v1/openAlgoOrders`
- Changed order type check from `'STOP_MARKET'` to `algoType == 'CONDITIONAL'`
- Updated field references to use `triggerPrice` and `algoId` for algo orders

**Files Modified:**
- `src/order_executor.py`
